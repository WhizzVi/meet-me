# Романтичное приглашение на свидание — дизайн

**Дата:** 2026-05-30
**Стек:** Next.js 15.3, App Router, TypeScript, чистый CSS, хранение в JSON-файле.

## Цель

Небольшое веб-приложение — игривое романтичное приглашение девушки на свидание, в нежно-пастельном «кошачьем» стиле. Девушка проходит сценарий из нескольких страниц; её выбор (дата/время и блюдо) сохраняется, а автор приглашения смотрит результат на защищённой паролем странице.

## Решения

- **Визуальный стиль:** нежно-пастельный (розово-персиковый), округлые кнопки, парящие сердечки, дружелюбный округлый шрифт.
- **Котики/графика:** эмодзи + inline-SVG (сердечки, лапки, котики). Без внешних файлов и зависимостей от интернета.
- **Хранение:** локальный JSON-файл `data/submissions.json` через Route Handlers.
- **Защита `/show`:** простой пароль (`SHOW_PASSWORD` в `.env.local`), проверка на сервере, сессия в куке.
- **Устройства:** телефон + ПК (адаптив, убегающая кнопка работает и от курсора, и от тапа).

## Страницы

| Путь | Назначение |
|---|---|
| `/` | Вопрос «Ты пойдёшь со мной на свидание?» + кнопки **Да** 💖 и **Нет** 😢. «Нет» при наведении/тапе → 😏 и убегает рывками. «Да» → `/when`. |
| `/when` | «Клааасс! ❤️ Когда у тебя есть свободное окошко?» 😼 + виджет выбора даты и времени → `POST /api/submit` → `/what`. |
| `/what` | Выбор блюда отдельными кнопками с иконками: рамен 🍜, морепродукты 🦐, паста 🍝, стейк 🥩, суши 🍣, борщ 🥣, десертики 🍰. Выбор → дописать в запись → `/final`. |
| `/final` | «Супер ❤️ Тогда я заеду за тобой [дата] в [время] и поедем кушать [блюдо]» + довольный котик 😻. |
| `/show` | Форма пароля → таблица всех попыток (дата/время свидания, блюдо, когда заполнено), новые сверху. |

## Поведение «убегающей» кнопки «Нет»

- Кнопка **не телепортируется**, а очень быстро (анимированно, ~0.15–0.25 с) уезжает в новую случайную точку контейнера — движение визуально заметно (CSS `transition: transform` + `ease-out`).
- **ПК:** при `mouseover`/приближении курсора кнопка резко скользит на новое место; смайлик меняется на 😏. Серия быстрых рывков, поймать почти нельзя.
- **Телефон:** при `touchstart`/`pointerdown` кнопка так же быстро уезжает при попытке тапа — тот же эффект.
- Кнопка «Да» неподвижна; по клику — переход на `/when`.

## Хранение и поток данных

Файл `data/submissions.json` — массив записей вида:

```
{ id, date, time, dish, createdAt, updatedAt }
```

Поток (каждый проход теста = новая попытка, старые не перезаписываются):

1. `/when` → `POST /api/submit { date, time }` всегда создаёт **новую** запись с `id` и `createdAt` (момент заполнения), ставит свежую куку `sid` на эту попытку.
2. `/what` → `POST /api/submit { sid, dish }` дописывает блюдо и обновляет `updatedAt` у записи с этим `sid`.
3. `/final` → `GET /api/submission?sid=` читает запись и подставляет дату/время/блюдо в текст.
4. `/show` после успешного пароля читает весь файл и рендерит таблицу всех попыток, отсортированную по времени (новые сверху).

Если девушка пройдёт тест несколько раз, в файле будет несколько записей — на `/show` видны все попытки с временем заполнения (`createdAt`/`updatedAt`).

API:

- `POST /api/submit` — создание записи (дата/время) или обновление (блюдо) по `sid`.
- `GET /api/submission?sid=` — одна запись для `/final`.
- `POST /api/show-login` — проверка пароля, выдача куки сессии.
- `GET /api/submissions` (или server-side в `/show`) — все записи, только при валидной сессии.

## Стиль и компоненты

- Палитра: розово-персиковые градиенты, мягкие тени, скругления.
- Переиспользуемые компоненты: фон с парящими сердечками, кнопка-«пилюля», карточка-блок текста, SVG-котики.
- Анимации: плавные появления, пульсация сердец, рывки убегающей кнопки — чистый CSS + минимум JS.

## Тестирование

- Запуск `npm run dev`, визуальная проверка каждой страницы через preview.
- Проверка сохранения в JSON и отображения на `/show`.
- Проверка убегающей кнопки на ПК (мышь) и в мобильном вью (тап).

## Вне рамок (YAGNI)

- Внешняя БД, аутентификация сложнее простого пароля, мультиязычность, деплой-конфиги, аналитика.
